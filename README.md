바이브 코딩 2주차 과제 공지합니다!

이번 주는 본격적인 코딩 전에, 주제를 구체화하는 단계입니다.
자유롭게 주제를 확장하시면 되고, 만약 이 과정에서 구현이 어렵다고 판단되면 자유롭게 주제나 기능을 변경하시면 됩니다. 

1. 기능 구체화:
1주차에 정했던 핵심 기능을 구체화하시면 됩니다.
[구체화 예시]
로그인 기능 -> 이메일/비밀번호 입력하여 로그인 + 실패시 에러 메시지 출력
시간표 표시 기능 -> 교수, 강의실, 수업명 표시 및 오늘 있는 수업 상단에 표시

2. 페이지 구성 구체화:
어떤 페이지들로 구성할지 생각하시면 돼요
ex) 로그인 페이지, 메인 페이지, 사용자 정보 페이지, 과제 관리 페이지

3. 기술 스택 결정:
프론트엔드만 구현할지 백엔드까지 구현할지 정한 뒤, 어떤 프레임워크 및 기술 스택을 사용할지 결정하시면 됩니다

프론트엔드: 겉으로 보이는 화면 및 내용을 담당하는 분야
ex) 웹 사이트 화면
백엔드: 보이지 않는 곳에서 정보 및 데이터를 관리하는 분야
ex) 사용자 정보 관리, 데이터베이스 저장

프론트엔드만 구현하는 것이 간단하지만, 제대로 된 사용자 데이터 관리나 로그인 기능을 구현하려면 백엔드까지 구현하는 것이 좋아요 다만 백엔드까지 구현하려면 난이도가 높아집니다

### 0. 로그인 화면
- UProvider / User 선택
- User 선택시 비회원 로그인 옵션 추가
- 이메일 / 비밀번호 회원가입/로그인 & Flutter 구글 회원가입

- 비밀번호 입력 오류시 "비밀번호가 잘못되었습니다. " 출력
